## 1. 产品概述

### 一句话定位
GESP Ace 是面向青少儿编程考级的 AI 智能陪练工具，通过"每日一练+智能诊断+个性化学习路径"帮助考生高效备考 GESP C++ 1-8 级。

### 目标用户

| 用户群体 | 核心需求 | 痛点 |
|---------|---------|------|
| **小学生/初中生**（主要用户） | 系统学习 C++ 编程，通过 GESP 认证 | 时间有限，需要高效学习；编程基础薄弱，需要即时反馈 |
| **家长**（付费决策者） | 了解孩子学习进度，验证学习效果 | 不懂编程，无法辅导；担忧孩子沉迷游戏化产品 |
| **教师/机构**（B端潜在用户） | 批量管理学生，布置作业，查看学情 | 缺乏针对性的题库和教学工具 |

### 核心价值主张

```
┌─────────────────────────────────────────────────────────┐
│  🎯 AI 诊断 × 个性化路径 × 精准刷题                      │
│                                                         │
│  "不做无用功，每次练习都指向提分"                         │
└─────────────────────────────────────────────────────────┘
```

- **AI 诊断**：错题不再是"做错了"，而是"错因归类+薄弱点定位"
- **个性化路径**：不是所有人都从一级刷到八级，是基于诊断的专属路线
- **精准刷题**：严格对齐 GESP 考纲，每题对应明确知识点

### 竞品差异化

| 维度 | 传统题库 | GESP Ace |
|-----|---------|---------|
| 题目来源 | 通用题库 | 严格对齐 GESP 考纲 |
| 错题处理 | 收藏+重做 | AI 错因诊断+变式题 |
| 学习路径 | 人工规划 | AI 动态生成 |
| 编程题 | 无或简单判题 | AI 代码审查+风格建议 |
| 进度呈现 | 正确率数字 | 知识点雷达图 |

**等级卡片状态：**

| 状态 | 条件 | 显示 |
|-----|------|-----|
| 🟢 已通过 | 历史正确率 ≥ 80% | "已掌握" + 徽章 |
| 🟡 进行中 | 有练习记录，正确率 40%-80% | 进度百分比 |
| 🔴 待加强 | 有练习记录，正确率 < 40% | "需加强" |
| 🔒 未解锁 | 前置等级未通过 | 锁图标 + 解锁条件 |

**功能逻辑：**

```python
# 等级解锁规则
def check_level_unlock(user_id: int, level: int) -> dict:
    """
    返回等级卡片状态和可访问性
    """
    user = get_user(user_id)
    current_level_data = get_level_progress(user_id, level)
    
    # 等级1始终解锁
    if level == 1:
        return {"unlocked": True, "status": "in_progress", "progress": 0}
    
    # 前置等级
    prerequisite = level - 1
    prereq_data = get_level_progress(user_id, prerequisite)
    
    # 前置等级正确率≥80%则解锁
    if prereq_data.correct_rate >= 0.8:
        return {
            "unlocked": True,
            "status": current_level_data.status,
            "progress": current_level_data.progress
        }
    else:
        return {
            "unlocked": False,
            "status": "locked",
            "requirement": f"需要先通过 {prerequisite} 级（正确率≥80%）"
        }
```

### 4.2 每日一练（核心功能）

**产品定位：** 用户粘性核心，每日必访入口。

**题目配置规则：**

```python
# 每日一练题目生成规则
DAILY_PRACTICE_RULES = {
    "total_questions": 3,
    "composition": [
        {"type": "choice", "count": 1, "points": 2},
        {"type": "judgment", "count": 1, "points": 2},
        {"type": "coding", "count": 1, "points": 25},
    ],
    "difficulty_adjustment": {
        # 基于用户近7天正确率动态调整
        "high_performance": "推荐当前等级-1难度",
        "normal": "推荐当前等级难度",
        "low_performance": "推荐当前等级+补强薄弱知识点"
    },
    "knowledge_coverage": {
        # 每天尽量覆盖不同的知识点
        "prefer_new_knowledge": 0.6,  # 60%新知识点
        "review_weak": 0.4          # 40%薄弱知识点复习
    }
}

#### 8.4.3 还原要点清单

| 维度 | 真实GESP考试特征 | 还原要求 |
|-----|-----------------|---------|
| **页面布局** | 左侧题目区 + 右侧答题卡 | 1:1还原，答题卡点击可跳转 |
| **题型导航** | 单选(15)→判断(10)→编程(2)，按顺序 | 同样顺序，同样题号编排 |
| **计时器** | 右上角倒计时，到时自动交卷 | 同样位置、同样逻辑、到时强制交卷 |
| **答题卡** | 已答●/未答○，颜色区分 | 同样视觉，点击跳转对应题目 |
| **选择题交互** | 点击选项即选中，高亮显示 | 同样点击交互，选中后答题卡同步标● |
| **判断题交互** | 正确/错误两个选项 | 同样交互 |
| **编程题** | 左侧题目+右侧代码编辑器 | 同样分栏，Monaco Editor或CodeMirror |
| **代码提交** | 运行→显示测试点结果 | 同样：先运行看部分测试点，提交看全部 |
| **交卷确认** | 二次确认弹窗 | 同样二次确认，显示已答/未答数量 |
| **考试结果** | 即时出分（选择+判断） | 选择判断即时出分，编程题AI评分 |
| **字体与配色** | 标准网页字体，白底黑字，简洁 | 尽量还原，不花哨 |

#### 8.4.4 与普通刷题模式的关系

```
两种模式共存：

1. 练习模式（日常使用）
   - 不计时，随意翻题
   - 每题做完即时AI解析
   - 适合学习和巩固

2. 全真模拟模式（终极版）⭐
   - 完全还原GESP考试界面
   - 严格计时，到时自动交卷
   - 考后统一出分+AI详细解析
   - 适合考前模拟和心理适应
   
   → 学生平时在"真实考试环境"里练
   → 考试当天界面完全熟悉，零紧张感
```

ACCOUNT_SETUP = {

> 本章节设计 GESP Ace 的智能录题系统，支持管理员上传 GESP 真题 PDF，AI 自动解析题目、匹配知识点、标注难度，大幅提升录题和维护效率。

---

### 13.1 功能概述

```
┌─────────────────────────────────────────────────────────────┐
│ 智能录题系统 · 一句话定位                                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  📄 上传 PDF → 🤖 AI 解析 → ✅ 预览确认 → 📚 一键入库       │
│                                                             │
│  管理员只需上传真题 PDF，系统自动完成题目提取、知识点匹配、    │
│  难度标注，管理员预览确认后批量入库，将录题效率提升 10 倍。    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 13.2 完整流程设计

```
┌──────────────────────────────────────────────────────────────────────┐
│                           智能录题流程总览                              │
├──────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐   │
│   │ 上传PDF  │────▶│ AI解析   │────▶│ 知识点   │────▶│ 难度    │   │
│   │ 拖拽/粘贴 │     │ 提取题目  │     │ 自动匹配  │     │ 自动标注  │   │
│   └──────────┘     └──────────┘     └──────────┘     └────┬─────┘   │
│                                                             │         │
│                                                             ▼         │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐   │
│   │ 返回首页  │◀────│ 一键入库  │◀────│ 预览确认  │◀────│ 来源标注  │   │
│   └──────────┘     └──────────┘     └──────────┘     └──────────┘   │
│                                                                        │
└──────────────────────────────────────────────────────────────────────┘
```

**详细步骤：**
1. **上传PDF**：支持拖拽/粘贴，格式 PDF/PNG/JPG，单文件≤20MB
2. **AI解析**：PDF转图片 → 多模态LLM识别 → 提取题目结构
3. **知识点匹配**：对照GESP 1-8级考纲（70+知识点）自动打标签
4. **难度标注**：基于知识点等级+复杂度分析（简单/中等/困难）
5. **来源标注**：自动解析文件名提取年份、季度、等级
6. **预览确认**：左侧PDF预览，右侧题目列表，支持逐题编辑
7. **批量入库**：仅入库已确认题目，自动去重

### 13.3 PDF解析技术方案

#### 推荐方案：多模态LLM识别

```
PDF文件 → PDF转图片（每页一张PNG，2x分辨率）→ GPT-4o/通义千问VL识别 → JSON结构化输出
```

**优点**：保留格式、识别代码/公式、适应GESP真题PDF（含表格、代码块、公式）
**缺点**：成本较高

```python
# PDF解析Prompt
PDF_PARSING_PROMPT = """
你是一个GESP编程考级真题解析专家。请从图片中提取所有题目信息。

【提取要求】
1. 选择题：题干 + 4个选项 + 正确答案
2. 判断题：题干 + 正确答案
3. 编程题：题干 + 输入输出样例 + 测试用例

2. 代码块处理：使用 ```cpp ... ``` 包裹，保持缩进
3. 返回JSON格式，包含每题的置信度
```

## 14. 题目展示格式规范 ⭐代码显示标准

> 本章节定义GESP Ace题目中代码的展示规范，确保C++代码在所有端都能正确、高质量展示。

---

### 14.1 代码块展示规范

```
┌─────────────────────────────────────────────────────────────┐
│ 【PC端完整代码块】                                           │
├─────────────────────────────────────────────────────────────┤
│  💻 代码                                                    │
│  ────────────────────────────────────────────────          │
│  1 │ #include <iostream>                             │   │
│  2 │ using namespace std;                            │   │
│  3 │ int main() {                                    │   │
│  4 │     int n;                                      │   │
│  5 │     cin >> n;                                  │   │
│  6 │     for(int i = 0; i < n; i++) {              │   │
│  7 │         cout << "Hello";                        │   │
│  8 │     }                                           │   │
│  9 │     return 0;                                   │   │
│ 10 │ }                                               │   │
└─────────────────────────────────────────────────────────────┘

【展示要求】
• 等宽字体：SF Mono / Fira Code / Consolas
• 语法高亮：关键字蓝色、字符串绿色、注释灰色
• 行号：左侧显示，固定宽度
• 深色背景：#1E1E1E
• 缩进：严格保留原始缩进（4空格）
```

---

### 14.2 特殊字符处理

```
┌─────────────────────────────────────────────────────────────┐
│ 必须正确展示的字符                                            │
├─────────────────────────────────────────────────────────────┤
│ 字符类型          │ 正确展示        │ 错误展示               │
│─────────────────────────────────────────────────────────────│
│ C++头文件        │ <iostream>      │ <iostream>（被HTML解析）│
│ 流运算符         │ cout << endl   │ cout < < endl         │
│ 逻辑运算符       │ && ||          │ && ||                 │
│ 字符常量         │ 'a'            │ 'a'                   │
│ 字符串常量       │ "hello"        │ "hello"               │
│ 转义字符         │ \n \t \\       │ \n \t \\              │
└─────────────────────────────────────────────────────────────┘

【处理方案】
• 存储：Markdown格式，使用```cpp围栏
• 渲染：<iostream>中的<>必须转义但保留<>样式
• 使用highlight.js进行C++语法高亮
```

---

### 14.3 代码样式分类

```
【1. 完整代码块】深色背景 + 语法高亮 + 行号 + 独立卡片

【2. 代码片段】深色背景 + 语法高亮 + 标注"代码片段"

【3. 行内代码】浅色背景pill样式（#F5F5F5）
   正确答案是 `cout << "Hello"` 语句。

【4. 伪代码】斜体 + 淡色边框 + 无语法高亮
```
